# Slice 18 — v0.1 hardening and release gate

## Goal

Prove the complete v0.1 product against its security, failure, accessibility, and workflow acceptance criteria.

## Dependencies

- [17-installer-upgrade-and-recovery](17-installer-upgrade-and-recovery.md)

## Non-goals

- No new product features.
- No v0.2 multi-user work.
- No performance claims without published measurements.

## Proposed files

- `tests/e2e/`
- `tests/failure/`
- `tests/security/`
- `tests/accessibility/`
- `docs/operators/`
- `docs/users/`
- `docs/security/`
- `CHANGELOG.md`
- `RELEASE.md`
- `.github/workflows/release.yml`

## Test-first implementation tasks

1. [ ] Turn every v0.1 specification exit gate into an executable acceptance test or an explicitly reviewed manual check.
2. [ ] Run full Sandbox, VPS, and Devbox journeys in container-only and KVM-enhanced environments.
3. [ ] Inject daemon, Incus, Caddy, DNS, disk, network-policy, and LLM upstream failures and verify truthful recovery states.
4. [ ] Run SSH fuzzing, API authorization, CSRF, WebSocket origin, route SSRF, gateway replay, redaction, and east-west isolation suites.
5. [ ] Run keyboard, screen-reader, contrast, responsive-layout, and terminal reconnect accessibility checks.
6. [ ] Publish measured create, start, clone, exec, and gateway-stream baselines without setting unsupported SLAs.
7. [ ] Complete threat-model review, dependency/license audit, signed-artifact pipeline, SBOM, checksums, and vulnerability-reporting path.
8. [ ] Write user quickstarts for VPS, Devbox/Pi, Sandbox API, optional domains, shared credentials, backup, and recovery.
9. [ ] Require a clean-host install, upgrade, backup, restore, and uninstall rehearsal before tagging v0.1.

## Verification

- [ ] `make check` plus container and KVM end-to-end matrices.
- [ ] Failure-injection and security suites.
- [ ] Installer/upgrade/restore rehearsals on every supported host target.
- [ ] Release artifact verification from a clean machine.

## Acceptance gate

- [ ] All specification exit gates have evidence.
- [ ] Known limitations clearly distinguish containers, VMs, local credentials, and broker-supported credentials.
- [ ] The release process can reproduce signed artifacts from a tagged source tree.

## Slice boundary

This slice is complete only when its tests, operator/developer documentation, and acceptance gate pass. Do not pull work from later slices merely because an interface leaves room for it.
